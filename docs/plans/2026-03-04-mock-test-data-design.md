# Mock 管理测试数据设计方案

> **创建日期**: 2026-03-04
> **当前版本**: v1.0
> **状态**: 已确认

---

## 一、目标

构造测试数据验证 Mock 管理模块的全部功能，包括前端界面验证，并保留测试数据供后续使用。

---

## 二、方案

**混合方案**：Python 脚本创建测试数据 + Playwright E2E 测试验证前端功能。

---

## 三、文件结构

```
backend/
└── tests/
    └── seed_mock_data.py       # Python 脚本：创建测试数据

frontend/
└── e2e/
    └── mock-management.spec.ts  # Playwright E2E：验证前端功能
```

---

## 四、测试数据设计

### 4.1 Mock 套件（5个）

| 套件名称 | 平台 | 启用 | 对比 | 规则数 | 响应数 | 白名单数 | 验证场景 |
|----------|------|------|------|--------|--------|----------|----------|
| Web登录Mock | web | ✓ | ✓ | 2 | 1 | 2 | 基础功能、对比 |
| APP首页Mock | app | ✓ | ✗ | 1 | 2 | 0 | 多响应、APP平台 |
| 用户中心Mock | web | ✗ | ✓ | 3 | 1 | 1 | 禁用状态、多规则 |
| 支付接口Mock | web | ✓ | ✗ | 0 | 1 | 3 | 无规则、多白名单 |
| 订单查询Mock | web | ✓ | ✓ | 2 | 3 | 1 | 完整配置 |

### 4.2 规则类型覆盖

- `equals`: 精确匹配
- `contains`: 包含匹配
- `not_equals`: 不等于

### 4.3 白名单类型覆盖

- `clientId`: 客户端ID
- `userId`: 用户ID
- `vid`: 访客ID

## 五、E2E 测试覆盖

| # | 测试场景 | 验证点 |
|---|----------|--------|
| 1 | 套件列表加载 | 显示 5 个套件，分页正常 |
| 2 | 按状态筛选 | 启用/禁用筛选正确 |
| 3 | 新建套件 | 表单填写、保存成功、列表更新 |
| 4 | 编辑套件 | 修改名称、规则、保存成功 |
| 5 | 复制套件 | 弹窗显示、新名称输入、复制成功 |
| 6 | 删除套件 | 确认弹窗、删除成功、列表更新 |
| 7 | 规则配置 | 添加/编辑/删除规则 |
| 8 | 响应配置 | 添加响应、JSON 编辑、预览 |
| 9 | 白名单配置 | 添加/删除白名单 |
| 10 | 对比记录查看 | 列表显示、详情弹窗 |

## 六、执行方式

```bash
# 1. 创建测试数据（保留在数据库中）
cd backend && python tests/seed_mock_data.py

# 2. 运行 E2E 测试验证前端
cd frontend && npx playwright test e2e/mock-management.spec.ts

# 3. 手动验证（可选）
# 打开前端页面，查看测试数据是否正常显示
```

---

## 七、验收标准

1. Python 脚本执行成功，数据库中有 5 个 Mock 套件
2. E2E 测试全部通过（10 个测试用例）
3. 前端页面可正常显示和操作测试数据
4. 测试数据保留在数据库中，可重复使用
